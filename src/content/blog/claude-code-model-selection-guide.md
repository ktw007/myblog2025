## 🚀 Claude Opus → GPT-5

### 模型信息

- **Claude Code 选择**：`Claude Opus`
- **实际模型**：GPT-5
- **推理模式**：**动态切换**（根据思考模式）

### 推理模式切换

Claude Code 的"思考模式"（Thinking Mode）会影响 GPT-5 的推理深度：

#### 💡 思考模式关闭（默认）

- **推理模式**：`gpt-5-low`（低推理深度）
- **切换方式**：默认状态，或按 `Tab` 键切换到"不思考"
- **特点**：
  - ⚡ 响应速度快
  - 📝 适合日常编码任务

#### 🧠 思考模式开启

- **推理模式**：`gpt-5-high`（高推理深度）
- **切换方式**：按 `Tab` 键切换到"思考模式"
- **特点**：
  - 🎯 深度推理能力
  - 🏗️ 复杂问题解决
  - ⏱️ 响应时间较长
  - 💎 Token 消耗较高

### 如何切换思考模式

在 Claude Code 对话框中：

1. **查看当前模式**：对话框底部显示"思考"或"不思考"
2. **切换模式**：按 `Tab` 键即可切换
3. **发送消息**：切换后再发送消息，推理模式自动生效

```
┌─────────────────────────────────────┐
│ 💬 输入你的问题...                   │
│                                     │
│ [Tab] 思考 ← → 不思考 [Tab]         │ ← 按 Tab 切换
└─────────────────────────────────────┘
```

### 适用场景

#### 使用 GPT-5-Low（思考模式关闭）

✅ 代码补全和建议
✅ 简单代码生成
✅ 日常问题解答
✅ 代码格式化

#### 使用 GPT-5-High（思考模式开启）

✅ **架构设计**：系统架构规划和设计
✅ **复杂问题**：多步骤推理、算法设计
✅ **技术决策**：技术方案对比和选择
✅ **深度分析**：代码审查、性能优化建议
✅ **创意编程**：非常规解决方案探索

### 性能对比

| 指标         | GPT-5-Low | GPT-5-High |
| ------------ | --------- | ---------- |
| **响应速度** | ⚡⚡⚡⚡⚡ 快  | ⏱️⏱️⏱️ 较慢   |
| **推理深度** | ⭐⭐⭐ 标准  | ⭐⭐⭐⭐⭐ 深度 |
| **适用任务** | 日常编码  | 复杂推理   |

## 🔍 常见问题

### Q: 如何知道当前使用的是哪个推理模式？

A: 查看 Claude Code 对话框底部：
- 显示"不思考" → 使用 GPT-5-Low
- 显示"思考" → 使用 GPT-5-High

### Q: 思考模式需要每次都切换吗？

A: 不需要。切换后的模式会保持，直到你再次按 `Tab` 键切换。

### Q: GPT-5-High 慢多少？

A: 通常响应时间是 GPT-5-Low 的 2-3 倍，但推理质量显著提升。

### Q: 如何节省 Token？

A: 建议策略：
- 注意上下文，越多请求一次消耗越多

### Q: 可以混合使用吗？

A: 可以！在同一会话中随时切换模型和推理模式。

### Q: 为什么选择 Opus 实际是 GPT-5？

A: 这是选择器的智能映射机制，为 Claude Code 用户提供更强大的推理能力，同时保持界面一致性（标准公关）。
B: opus太贵。

> 💡 提示：实际消耗取决于代码复杂度、上下文长度等因素，且claude code内置大量的系统提示词以及工具（不可删，删了就废了），添加mcp工具也会占用token消耗，所以mcp最好用到再加。

> 💡 提示：思考模式也会额外消耗，建议claude 4.5就没必要开启了，效果差不多，开启了速度还慢了，该错的还会错。

> 💡 提示：gpt-5-high 项目越大时间阅读，测试使用重构一个项目方案花了差不多10分钟，才给初步方案

> 💡 提示：当前会话反复给出不合适的答复，建议开启新会话排除上下文污染，否则很可能继续错下去（切模型效果不大，因为上下文影响）

## 🎯 最佳实践

### 1. 任务分类使用

```
┌──────────────────┐
│  快速代码生成    │ → Claude Sonnet 4.5
├──────────────────┤
│  创意性编码      │ → GPT-5-Low
├──────────────────┤
│  复杂架构设计    │ → GPT-5-High
└──────────────────┘
```

### 2. 渐进式提升

从简单到复杂：
1. 先用 Claude Sonnet 4.5 尝试
2. 如需更多创意，切换到 GPT-5-Low
3. 遇到难题时，开启思考模式使用 GPT-5-High

### 3. Token 优化策略

- ✅ 日常任务优先 Claude Sonnet 4.5（快且省）
- ✅ 仅在需要时使用 GPT-5-High（慢但强）
- ✅ 避免在简单任务上使用高推理模式
